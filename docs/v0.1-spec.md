# Green v0.1 Specification

## Goal

Confirm whether a Windows agent is installed and running by sending a heartbeat to an admin server every 60 seconds.

## Status Rules

- Active: last heartbeat is less than 3 minutes old.
- Delayed: last heartbeat is between 3 and 10 minutes old.
- No Signal: last heartbeat is more than 10 minutes old.
- Never Connected: device is registered but no heartbeat has arrived.

## First Test Flow

1. Start the FastAPI server.
2. Open the dashboard.
3. Register a device with the recovery name only.
4. Copy the generated activation token from the dashboard.
5. Start the GUI agent.
6. Enter the activation token and activate the device.
7. Confirm that the dashboard turns the device green.
8. Stop the agent and watch the dashboard move to delayed and then no signal.

## API

### `POST /api/heartbeat`

Request:

```json
{
  "device_id": "green-test-001",
  "token": "change-me-12345",
  "device_name": "DESKTOP-123",
  "windows_user": "user",
  "agent_version": "0.1.0",
  "status": "running"
}
```

### `POST /api/activate`

Request:

```json
{
  "enrollment_token": "generated-token-from-dashboard",
  "device_name": "DESKTOP-123",
  "windows_user": "user",
  "agent_version": "0.1.0"
}
```

Response:

```json
{
  "ok": "true",
  "message": "activated",
  "device_id": "green-abc123def456",
  "token": "generated-token-from-dashboard",
  "recovery_name": "Person or case code"
}
```

Response:

```json
{
  "ok": "true",
  "message": "heartbeat accepted"
}
```
